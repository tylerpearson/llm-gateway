llm-gateway has been deployed.

1. Get the gateway URL:
{{- if .Values.ingress.enabled }}
{{- range $host := .Values.ingress.hosts }}
  http{{ if $.Values.ingress.tls }}s{{ end }}://{{ $host.host }}
{{- end }}
{{- else if contains "NodePort" .Values.service.type }}
  export NODE_PORT=$(kubectl get --namespace {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services {{ include "llm-gateway.fullname" . }})
  export NODE_IP=$(kubectl get nodes --namespace {{ .Release.Namespace }} -o jsonpath="{.items[0].status.addresses[0].address}")
  echo http://$NODE_IP:$NODE_PORT
{{- else if contains "LoadBalancer" .Values.service.type }}
  NOTE: It may take a few minutes for the LoadBalancer IP to be assigned.
  export SERVICE_IP=$(kubectl get svc --namespace {{ .Release.Namespace }} {{ include "llm-gateway.fullname" . }} --template "{{"{{range (index .status.loadBalancer.ingress 0)}}{{.}}{{end}}"}}")
  echo http://$SERVICE_IP:{{ .Values.service.port }}
{{- else }}
  kubectl --namespace {{ .Release.Namespace }} port-forward svc/{{ include "llm-gateway.fullname" . }} 8080:{{ .Values.service.port }}
  echo "Gateway available at http://127.0.0.1:8080"
{{- end }}

2. Check liveness / readiness:
  kubectl --namespace {{ .Release.Namespace }} exec deploy/{{ include "llm-gateway.fullname" . }} -- wget -qO- http://localhost:8080/healthz
  kubectl --namespace {{ .Release.Namespace }} exec deploy/{{ include "llm-gateway.fullname" . }} -- wget -qO- http://localhost:8080/readyz

3. Prometheus metrics are exposed at /metrics (port 8080) and scraped
   automatically when Prometheus is configured to use pod annotations.

4. To update provider API keys without redeploying, edit the Secret:
   kubectl --namespace {{ .Release.Namespace }} edit secret {{ include "llm-gateway.secretName" . }}
   Then restart the deployment to pick up the new values:
   kubectl --namespace {{ .Release.Namespace }} rollout restart deploy/{{ include "llm-gateway.fullname" . }}
