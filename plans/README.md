## Findings considered and rejected

- Auth cache map delete-during-range (reported as a correctness bug): deleting
  map entries during range iteration is defined behavior in Go and the sweep
  runs under the mutex. Not a bug.
- Per-event unmarshal/marshal in translate streaming: inherent to shape
  translation; a byte-level rewrite would add risk for marginal gain.
- Discarded write errors on final translate SSE events: the last write's
  error is returned, so an interrupted stream still surfaces; the discarded
  intermediates are practically unobservable. Not worth doing.
- Bounded in-memory capture buffer for caching: deliberate bounded design
  (cache.MaxBytes); revisit only with production heap profiles.
- setModel full unmarshal/marshal: uses json.RawMessage already; micro-opt.
- Config struct decomposition: flat nested config structs are idiomatic Go;
  loaded once at startup.
- Missing tool_use/image translation tests: translate is documented as
  text-only for v1 (package comment), so absent coverage of unsupported
  features is not a gap; see the deferred hardening item instead.
- Pre-commit hook template: CI enforces the gates; opt-in hooks add little.
- govulncheck and dependency posture: clean, nothing to do.
